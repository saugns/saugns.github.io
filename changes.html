Syntax changes

Below is a shorter summary of only the changes that impact how scripts in the language are written. This list may be updated ahead of new versions being tagged. The oldest tagged version v0.3.0 is compared to the historical (last 2012) sgensys-20120401.tgz version.

v0.3.11

  • For modulation with value range, change delimiters between the parts from X,Y~[...] to Xw,Y[...].
  • For phase modulation (normal and frequency-amplified) remove + after p and place to set phase value.
  • Rename \number subshift to ;number gapshift, since the two ; compound step syntaxes go together.
  • Newlines no longer end steps at the top scope, the new rules simpler and more consistent.
  • Ramp syntax.
    • Rename v for goal value to g.
    • Rename c before ramp name to r.
    • Rename ramp cos to sin (and use more accurate polynomial).
  • Numerical expressions.
    • Allow arithmetic and number signs outside parentheses. (Undoes v0.3.0 change, restoring old feature.)
    • Add % remainder operation. Make ^ (to the power of) right-associative.
    • Add functions cos(x), rand(), rint(x), seed(x), sin(x), time().
    • Add constants mf, pi.

v0.3.10

  • Timing syntax. (Also a big-issue fix for ;.)
    • Rename forward-shift \ to /. Remove special value with literal t.
    • Replace s silent time padding parameter with new, more flexible \ subshift syntax.
    • Make | reset, not add, delay to add for next step to prior duration.
    • Remove special value for t parameter with literal *.
  • Add phase named constant G for golden angle.

v0.3.10b

  • Add new p+f[...] syntax, for frequency-linked PM.

v0.3.9

  • Wave types.
    • Rename sha to ahs, szh to hrs, and ssr to srs.
    • Add ssr.
    • No longer use rounded corner shaped sqr, saw. Use new oscillator with DPW-ish aliasing reduction.
  • Ramp types.
    • Rename esd to xpe, and lsd to lge.
    • Add cos.
  • Numerical expressions.
    • Add functions abs(x), exp(x), log(x), met(x), sqrt(x).

v0.3.8

  • Nesting syntax. Rename undocumented generic [...] to <...>, still only for testing and further development.

v0.3.7

  • Replace panning P keyword with channel mixing parameter c and S c default, with named constants C, L, R.

v0.3.6

  • Nesting syntax. Add undocumented generic subscope [...] for testing and further development.

v0.3.3

  • Wave types. Remove shh. Use rounded corner shaped sqr, saw.

v0.3.2

  • Comment syntax. Replace line comment # with #!, remainder of file comment Q with #Q.

v0.3.1

  • AM and FM modulator list syntax, and related parameter values. Expressions like fX f!Y[...] change to fX,Y~[...].

v0.3.0

  • Nesting syntax.
    • List of operators and steps for them. From inside <...>, change to inside [...].
    • Compound value (e.g. ramp arguments). From inside [...], change to inside {...}.
    • PM modulator list prefix. From p!, change to p+.
  • Label syntax.
    • Names can now only contain alphanumeric characters and _.
    • Referencing syntax :name changed to @name.
  • Numerical expressions.
    • Only allow arithmetic and number sign within parentheses. (This went along with big fixes and more changes.)
  • Amplitude handling.
    • Add downscaling of output amplitude by voice count, enabled by default. Using S a disables it.
    • Allow a (amplitude) parameter for AM and FM modulators.
  • Disable multiplicative inverse for r (relative frequency).
  • Ramp types.
    • Renamed exp to esd, and log to lsd.
    • Added changed exp, log, and new hold.
  • Wave types.
    • Renamed srs to ssr.
    • Added sha, szh, shh.
  • Comment syntax.
    • Also recognize C-style /*...*/ and C++-style // comments.
